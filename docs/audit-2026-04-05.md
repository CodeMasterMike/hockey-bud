## Critical (Fix Immediately)

### SEC-001: Dev Credentials in Base appsettings.json — FIXED
- `backend/src/HockeyHub.Api/appsettings.json` had SQL Server password `HockeyHub_Dev1!` in the base config
- `docker-compose.yml` also has the password (acceptable — local Docker only)
- `secrets/` directory was never committed (already in `.gitignore` since project start)
- **Fix applied:** Moved connection strings from `appsettings.json` to `appsettings.Development.json` so the base config shipped to deployed environments has no credentials

### SEC-002: Unauthenticated Hangfire Dashboard — FIXED
- `backend/src/HockeyHub.Api/Program.cs` mapped `/hangfire` with no authorization
- Anyone could view, modify, and delete background jobs in deployed environments
- **Fix applied:** Dashboard now only mapped in Development environment; deployed environments (Staging/Production) do not expose it

### SEC-003: Public SQL Server in Azure — DOCUMENTED
- `infra/main.bicep` — dev SQL Server uses `publicNetworkAccess: 'Enabled'` with Azure Services firewall rule
- This is dev-only (`if (isDev)`) — prod uses a separately managed SQL Server via `sqlConnectionString` parameter
- Container Apps require this firewall rule since they don't have VNet integration on basic tier
- **Fix applied:** Added documentation comments clarifying dev-only scope and that prod should use Private Endpoints

### SEC-004: TrustServerCertificate=true — NOT AN ISSUE
- Only present in `appsettings.Development.json` for local Docker SQL container (no real cert)
- Deployed connection string in Bicep Key Vault secret already has `TrustServerCertificate=False` and `Encrypt=True`
- No action needed

### SEC-005: Dockerfile USER $APP_UID Undefined — FIXED
- `backend/Dockerfile` referenced undefined `$APP_UID` environment variable
- Container would fail to switch to non-root user
- **Fix applied:** Changed to `USER app` (built-in non-root user in .NET container images)

## High Severity

### SEC-005: No Content Security Policy Header — FIXED
- `frontend/staticwebapp.config.json` — Good security headers present (HSTS, X-Frame-Options DENY, nosniff) but no CSP
- **Fix applied:** Added restrictive CSP: `default-src 'self'`, `frame-src 'none'`, `object-src 'none'`, allows `wss:` for SignalR. Removed deprecated `X-XSS-Protection`.

### SEC-006: No Rate Limiting on API
- `backend/src/HockeyHub.Api/Controllers/ScoresController.cs` — All 5 public endpoints have no rate limiting
- **Action:** Add `Microsoft.AspNetCore.RateLimiting` middleware

### SEC-007: Redis Without Auth or Encryption
- `infra/main.bicep:195-222` — Redis container app has no password, no TLS
- Internal network access gets full cache access; potential cache poisoning
- **Action:** Add `--requirepass` flag; for prod use Azure Cache for Redis

### SEC-008: Unvalidated URL in bypassSecurityTrustResourceUrl — FIXED
- `frontend/src/app/components/shared/video-modal/video-modal.ts` — Video URL passed directly without validation
- **Fix applied:** URL now validated with `new URL()` parsing; rejects non-HTTPS and malformed URLs before bypassing sanitizer

### SEC-009: No Backend Security Headers Middleware — FIXED
- `backend/src/HockeyHub.Api/Program.cs` — No global security headers on API responses
- **Fix applied:** Added inline middleware setting X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy on all API responses

### SEC-010: No CI Security Scanning — FIXED
- `.github/workflows/ci.yml` — No dependency vulnerability scanning
- **Fix applied:** Added `dotnet list package --vulnerable` to backend job and `npm audit --audit-level=high` to frontend job

### REL-001: Silent Failures in Hangfire Sync Jobs — FIXED
- `ScoresSyncJob` and `StandingsSyncJob` — Null checks returned silently without logging
- **Fix applied:** Added `LogError` for missing league/season (with actionable "has DataSeed been run?" message) and `LogInformation`/`LogWarning` for empty API responses

### REL-002: SignalR invoke() Calls Have No Error Handling — FIXED
- `frontend/src/app/services/signalr.service.ts` — invoke calls had no try-catch
- **Fix applied:** All three invoke calls wrapped in try-catch with `console.error` logging

### REL-003: Database Migration Without Rollback
- `.github/workflows/deploy-prod.yml:95-102` — `dotnet ef database update` with no rollback on failure
- **Action:** Add migration validation step and rollback mechanism

### INFRA-001: ACR Basic SKU — No Image Scanning
- `infra/main.bicep:84` — Basic SKU lacks vulnerability detection and content trust
- **Action:** Use Standard or Premium SKU for production

### INFRA-002: SQL Server Missing Threat Protection & Auditing
- `infra/main.bicep:150-191` — No Vulnerability Assessment, Advanced Threat Protection, or Auditing configured
- **Action:** Add `securityAlertPolicies` and `auditingSettings` resources

## What's Working Well

- **Scores MVP is solid** — 5 endpoints, full UI, real-time updates via SignalR + Redis
- **Standings page** — 4 view modes (wildcard/division/conference/league), responsive layout (side-by-side wide, tabbed narrow), sortable columns, wild card cut-line semantics, 1h Redis cache
- **Schedule page** — ScheduleSyncJob populates full season from NHL API, monthly game list with navigation, 6h cache
- **Global search** — banner input with 300ms debounce, live dropdown grouping Players + Teams results
- **Teams pages** — teams index (4-column card grid with standings), team profile (header, detail cards, full roster table)
- **Game Hub** — team stats + player stats tabs, period box scores, team comparison, goals/penalties timeline with PPG/SHG/EN tags, skater/goalie stat tables
- **Trades page** — Trade + TradeAsset entities with daily sync, chronological trade cards with team logos and asset lists
- **Playoff bracket** — bracket + matchup detail via NHL API `/v1/playoff-bracket/{season}` with Redis cache, conference filtering, series score formatting; zero new entities
- **Draft page** — draft picks by round via NHL API `/v1/draft/{year}`, player ID resolution from DB, live draft day support (10s cache), round tabs with clickable player names
- **Season mode** — auto-detects regular-season/playoffs/off-season from game data, drives dynamic nav bar links
- **Playwright e2e tests** — 87 tests across desktop + mobile (7 spec files, 3 page objects) covering navigation, standings, teams, search, schedule, trades, game hub. Runs against deployed SWA or auto-starts ng serve locally
- **Modern Angular patterns** — OnPush everywhere, signals, takeUntilDestroyed, new control flow
- **Strict TypeScript** — `strict: true`, no `any` types in codebase
- **Clean architecture** — Core → Data → Api dependency flow properly enforced
- **Good security headers** (minus CSP) in Static Web App config
- **No observable memory leaks** — proper subscription cleanup throughout
- **Real-time infrastructure** — SignalR + Redis backplane + GameClockService with rAF
- **Full IaC coverage** — all Azure resources in Bicep with environment parameters
- **CI/CD pipeline** — PR gates, auto-deploy dev, manual prod with approval
